software

  • tea for plan9 (Tutorial).
Included in tea are a set of commands that implement temporal database operations:
  • tdbjoin
  • tdbadd
  • tdbcoin

tdbjoin and tdbadd are built using some Plan 9 libraries. A unix port of these libraries can be downloaded from here.

  • tea for unix, which includes a unix version of 9ls

The tea script is an application of the tdb tools aimed at meta data for a filesystem. That is annotating files with information about what they contain and when you used them, or how you used them. This is a poor man's version of Gordon Bell's MyLifeBits Project.

The tdb* tools have other uses also. I use them to index all my text files as the backend of a search engine. I use them for project tracking, time tracking and process control. I use them as a temporal replacement for the /rdb UNIX tools.

PostDate

PostDate is a language for managing time varying data and code.

PostDate's syntax is a postfix (reverse Polish) notation loosely based on PostScript. It uses bi-temporal (transaction time and valid time) binary relations to store and query time varying data.

Built in operators retrieve temporal triads of time, element, and attribute values from named relations. The value associated with a key may vary with time, and so the value retrieved depends on the choice of effective valid time. The value for a given key may contain PostDate code, which allows procedure definitions to vary along the valid time line. The bi-temporal nature of the database also allows us to set the effective transaction time and review the state of the database exactly as it appeared on some past date.
